# 熱GeoPT 研究計画書
## Thermal GeoPT: Lifted Geometric Pre-Training for Heat Transfer Surrogate Models

**目的：** GeoPTの事前学習戦略を熱解析サロゲートに転用し、論文としての成果にまとめる  
**実行環境：** NVIDIA RTX 5070 Ti（個人所有）  
**想定投稿先：** NeurIPS / ICLR / ICML（ML系）or IJHMT / Applied Thermal Engineering（熱工学系）

---

## 研究の位置づけと新規性

### 背景

GeoPT（MIT・清華大学、2026）は、CFDソルバーを使わずにSTLファイルから合成ダイナミクスデータを生成し、流体サロゲートモデルの事前学習に活用する手法を提案した。本研究はこの発想を**熱解析サロゲート**に初めて転用する。

### 新規性

1. **Thermal Lifted Geometric Pre-Training**の提案
   - ランダム熱流束場 ＋ 拡散トラジェクトリによる合成事前学習データ生成
   - ソルバー不要・STLのみで大量の事前学習データを生成可能

2. **Thermal Distance Field（TDF）**の定義
   - GeoPTのVDFに対応する熱版特徴量
   - 境界面からの「熱的距離」を形状のみから計算

3. 流体・熱・構造の3領域にわたるGeoPT拡張の**先駆的実証**

## 手法：Thermal GeoPT

### 熱版合成ダイナミクスの生成

#### ①Thermal Distance Field（TDF）の定義

VDFの熱版として、各空間点xから最寄り境界面への**熱的距離**を定義する：

```
TDF(x) = 境界面への最短距離 × 材料熱伝導率の重み
```

計算はVDF同様、ray-triangle intersectionのみで完結（ソルバー不要）。

#### ②ランダム熱流束場のサンプリング

境界面の各点に対してランダム熱流束をサンプリング：

```
q ~ Unif(B^C),  B^C = {q ∈ R : |q| ≤ q_max}
```

1形状あたり100通りの熱流束パターンを生成（GeoPTと同様）。

#### ③拡散トラジェクトリの計算

熱伝導方程式の離散化：

```
∂T/∂t = α∇²T  （熱拡散方程式）
```

ランダムウォーク近似で拡散軌跡を計算（移流方程式のトラジェクトリに対応）：
- 各空間点から隣接点へのランダムウォーク（τ=3〜5ステップ）
- 境界面到達で停止（断熱境界 = 熱流束ゼロ）

#### ④事前学習タスク

- **入力**：空間点座標 x₀ ＋ ランダム熱流束 q
- **予測**：各ステップのTDF値の時系列 {TDF(x₁), TDF(x₂), TDF(x₃)}
- **損失**：MSE（幾何計算による正解TDFとの比較）

### GeoPTとの対応関係

| GeoPT（流体） | Thermal GeoPT（熱） |
|---|---|
| Vector Distance Field（VDF） | Thermal Distance Field（TDF） |
| ランダム速度場 v | ランダム熱流束 q |
| 粒子トラジェクトリ | 拡散トラジェクトリ（ランダムウォーク） |
| 移流方程式（双曲型） | 熱伝導方程式（放物型） |
| 粒子が境界で停止 | 断熱境界で熱流束ゼロ |

## 事前学習データの収集方針

### 基本方針

事前学習には**STLのみ**が必要（CFD結果不要）。個人レベルで収集可能な以下のソースを組み合わせる。

### ソース①：ShapeNet（主力・無料）

| カテゴリ | 形状数 | 熱解析への関連性 |
|---|---|---|
| 航空機 | 4,045 | ✅ 翼型・外表面熱伝達 |
| 自動車 | 7,479 | ✅ エンジン冷却・外部熱流 |
| 船舶 | 1,939 | ✅ 外表面熱移動 |
| ランプ・電子機器 | 〜2,000 | ✅ **放熱・冷却設計に近い** |

熱解析の観点では、GeoPTが使った形状カテゴリに加え、**電子機器・ランプ類**が特に有効（放熱フィン形状が含まれる）。

### ソース②：自前CadQuery生成形状（補強）

| タイプ | 形状数 | 目的 |
|---|---|---|
| ポンプ Type A/B/C | 300形状 | ターゲットドメインとの近さ |
| 放熱フィン（板状・ピン状） | 200形状 | 熱解析の基本形状 |
| 簡易翼型（2D押し出し） | 100形状 | 翼面熱伝達の基礎 |

CadQueryで自動生成するため追加コストほぼゼロ。

### ソース③：GrabCAD（補助・任意）

放熱器・ヒートシンク・熱交換器などの形状。  
ライセンス確認が必要なため、研究目的での使用を明示した上で選定。

### 事前学習データ規模の見積もり

| ソース | 形状数 | ×100熱流束 | サンプル数 |
|---|---|---|---|
| ShapeNet（選定分） | 〜10,000 | ×100 | 100万 |
| 自前生成 | 600 | ×100 | 6万 |
| **合計** | **〜10,600** | | **〜106万** |

RTX 5070 Tiで処理可能な規模：GeoPTの134万サンプルに近く、十分な事前学習が期待できる。

## 実験環境・モデル設定

### RTX 5070 Tiでの制約と対応

| 制約 | 対応 |
|---|---|
| VRAM（推定16GB） | モデルサイズはBase（3M）またはLarge（7M）に限定 |
| ストレージ | 事前学習データを段階的に生成・使用 |
| 計算時間 | 事前学習を分割実行（週次） |

### バックボーン

Transolver（GeoPTと同一）をベースとして使用。  
事前学習後にファインチューニングで有効性が確認された後、GeoTransolverへの拡張を検討。

## 対照実験設計

### 実験グループ

| グループ | 設定 | 目的 |
|---|---|---|
| **A（提案手法）** | Thermal GeoPT事前学習 → ファインチューニング | 本手法の有効性検証 |
| **B（ベースライン①）** | Transolver ゼロから学習 | 事前学習なしとの比較 |
| **C（ベースライン②）** | GeoPT（流体）事前学習 → 熱ファインチューニング | 流体事前学習の転移効果比較 |
| **D（アブレーション①）** | TDFなし（VDFのみ）で熱事前学習 | TDFの効果検証 |
| **E（アブレーション②）** | ランダムウォークなし（静的TDFのみ）で事前学習 | 拡散トラジェクトリの効果検証 |

### グループCの意義

GeoPTの流体事前学習済み重みをそのまま熱ファインチューニングに使った場合の効果を測る。これにより「熱専用の事前学習が本当に必要か」という問いに答えられる。

### 評価指標

| 指標 | 内容 |
|---|---|
| Relative L2誤差 | 温度場予測精度（メイン指標） |
| 最高温度誤差 | 設計上最重要な指標 |
| 必要サンプル数 | 同精度達成に必要なCFDデータ量 |
| 収束エポック数 | 学習効率 |

### ファインチューニングデータ

OpenFOAMの`chtMultiRegionFoam`で生成した温度場VTPを使用。  
サンプル数を段階的に変化（10 / 50 / 100 / 500件）させ、**データ効率曲線**を描く。

## 実験フロー

```
【Step 1：TDF計算の実装】（1〜2週間）
  STL → ray-triangle intersection → TDF実装
  単純形状（球・立方体）で正解と比較・検証

【Step 2：合成データ生成パイプライン構築】（1〜2週間）
  ランダム熱流束サンプリング → 拡散トラジェクトリ計算
  1形状×100パターン生成の自動化
  ShapeNet形状を100形状でパイロット実行

【Step 3：Thermal GeoPT 事前学習】（2〜4週間）
  RTX 5070 Tiで事前学習を実行（Baseモデルから開始）
  チェックポイントを記録しながら収束を確認

【Step 4：ファインチューニング用データ生成】（並行・随時）
  OpenFOAM chtMultiRegionFoamで温度場VTPを生成
  ポンプ形状（Type C）＋ 翼型（外部熱伝達）で各100ケース

【Step 5：対照実験の実行】（3〜4週間）
  グループA〜Eを同条件で学習・評価
  データ効率曲線を描画

【Step 6：論文執筆】（4〜6週間）
  Results：対照実験比較・アブレーション
  Discussion：熱方程式の放物型特性への対応
  Ablation：TDF・拡散トラジェクトリ各要素の寄与
```
